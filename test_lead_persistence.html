Lead Persistence Test

Test Instructions

  1. Open the CRM application at http://localhost:3000
  2. Login as an agent
  3. Create a new lead
  4. Watch the browser console for auto-refresh logs
  5. Verify the lead persists for at least 60 seconds

Expected Behavior

  • ✅ Lead should appear immediately after creation
  • ℹ️ Console should show "Lead created, timestamp set to: [timestamp]"
  • ℹ️ Console should show "Skipping auto-refresh due to recent lead creation" for 60 seconds
  • ✅ Lead should persist for at least 60 seconds
  • ℹ️ After 60 seconds, auto-refresh should resume normally

Console Logs to Look For

Auto-refresh check: {timeSinceLastLeadCreation: 0, lastLeadCreationTime: 1234567890, shouldRefresh: false}
Skipping auto-refresh due to recent lead creation

Lead created, timestamp set to: 1234567890

If Issues Persist

  • Check if the backend is running on port 8000
  • Check if the frontend is running on port 3000
  • Check browser console for any errors
  • Verify the lead is actually created in the database
